# 卡价

这是一个以本地 HTML 为主的卡牌库存与售出管理项目。

当前为了兼容现有页面里的相对路径，网页入口文件和会被浏览器直接读取的资源仍然放在仓库根目录；其余脚本、说明和数据已经按用途拆分到子目录里，后续维护会轻松很多。

## 目录说明

```text
卡价/
├── index.html              # 主页面入口
├── tab0754.js              # 0754 售出记录组件

├── 卡价.xlsx               # 页面会直接读取的 Excel 文件
├── cardimg/                # 本地图像资源
├── supabase/               # Supabase Edge Functions

├── docs/                   # 项目说明和参考资料

```

## 页面功能

- “截图识别新增”支持选择图片，也支持在弹窗打开后直接粘贴剪贴板图片，再用 AI 识别或本地 OCR 识别。
- 卡牌编辑和新增时可以勾选 `PSA10` 标识；卡牌名称旁会显示 `PSA10` 标签。
- 有库存只卖出一部分时，不需要把整条记录手动改为“已售出”；只要在编辑弹窗里添加售出记录，系统会按 `数量 - 已售数量` 自动显示“部分售出”，并继续留在未售出/在库筛选中。
- 已售总额、已售成本和利润会按实际已售张数统计；部分售出的成本会按已售数量占总数量的比例分摊，避免整条库存提前进入已售统计。
- 表格“单价”下方会显示单价与参考价的差值：普通卡对比“流通品价”，勾选 `PSA10` 的卡对比“PSA10价”。
- 主表格每行新增“上架”按钮，会按卡片编号和语言准备集换社上架流程：先搜索对应卡片并进入详情，点击“出售”，普通卡默认选择“非评级卡 + 流通品相”，`PSA10` 默认选择“评级卡 + PSA”，再填入上架价格、备注、上传商品图，最后在集换社里点击“添加”完成上架。上架价格可使用参考价、填写具体价格，或按参考价加价百分比计算；上架备注默认留空，可按商品保存专用备注；上架弹窗底部只保留“保存设置”和“上架”，Pokopia 皮肤的操作列宽度也已适配“上架 / 编辑 / 删除”三个按钮。
- 集换社自动上架会把网页商品里的图片作为“上传商品图”自动加入：支持 `cardimg/` 本地图片、内嵌 base64 图片、绝对路径图片，以及可访问的网络图片。定价超过 500 元但商品没有图片时，网页会先提示补图，helper 也会停止流程，避免没有上传实物商品图就继续点“添加”。
- 主表格勾选多条商品后可以点击“批量上架已选”，系统会按已选队列逐个调用同一套集换社自动上架流程；批量弹窗里每个商品都可以单独修改上架价格，并提供“启动/检测本地服务”按钮来拉起和检查 `8767` 上架 helper。默认跳过已有“已上架”标签的商品，成功一条就给对应商品打上“已上架”标签，遇到失败或关闭“自动点击添加”时会停在当前商品，不会继续处理后面的队列。
- 自动上架推荐用 `python3 scripts/dashboard_server.py` 打开本地页面。网页会先检测 `http://127.0.0.1:8767/health`，如果上架 helper 没启动，会通过 `http://127.0.0.1:8000/start-jhs-listing-helper` 自动运行 `python3 scripts/jhs_listing_helper.py`，再调用 `http://127.0.0.1:8767/listings` 控制可见的集换社 App；“自动点击添加完成上架”默认勾选，会在填好页面后点击最终“添加”按钮，取消勾选才会停在“添加商品”页让你手动确认。如果页面提示 `Failed to fetch`、“本地上架服务未启动”或批量弹窗显示启动失败，说明页面不是通过 `dashboard_server.py` 打开的，或浏览器连不到本机 8000/8767 端口。
- 集换社上架搜索时，语言为“其他”不会强制选择集换社游戏筛选，只按卡片编号搜索并匹配结果；如果指定了简中/日文/英文等语言但临时找不到游戏筛选按钮，helper 会先返回搜索页并重新搜索一次，再继续后续流程。
- helper 重新搜索时只会识别集换社顶部真正的搜索框，不会把“添加商品”页里的商品价格或备注输入框误当成搜索框；如果当前停在添加页，会先返回到卡片详情，再继续返回到搜索页后重新搜索。
- 如果当前集换社停在商品管理、个人页、异常页面或找不到搜索入口，helper 会自动重启集换社 App，并点击底部“探索”回到搜索入口后重新搜索卡片。
- 自动上架依赖 macOS 辅助功能权限；如果提示 Python/终端没有权限，需要到“系统设置 -> 隐私与安全性 -> 辅助功能”同时允许 `Terminal` 和当前 Python，批量上架弹窗会提供“打开辅助功能设置”按钮辅助跳转。页面服务启动上架 helper 时会生成 `data/start_jhs_listing_helper.command` 并通过 Terminal 拉起 Python，以继承 Terminal 的辅助功能权限；helper 的 `/health` 会返回当前 `executable` 和 `accessibilityTrusted`。权限正常但批量上架开始时系统临时读不到集换社无障碍窗口时，helper 会先等待窗口出现，再自动重启集换社 App 重试一次；如果重启后仍读不到，会提示检查集换社窗口是否可见以及辅助功能权限。
- 勾选“自动点击添加完成上架”并成功提交后，页面会自动关闭上架弹窗、提示“上架完成”，并在对应商品名称旁显示“已上架”标签；标签里的 `×` 可以取消标记。如果默认停在集换社添加页由你手动点击“添加”，网页弹窗里也提供“已手动添加，标记已上架”按钮。
- 上架 helper 填写“商品价格”和“商品备注”时会优先使用集换社页面里的无障碍输入框节点；如果价格没有写入成功，会停止流程并提示错误，避免触发“商品价格没有填，自动设置为集换价？”后误按集换价提交。
- 上架 helper 处理多张库存时会定位“库存数量”的真实输入框，写入要上架的数量并读回确认；如果数量没有从 `1` 成功调整到商品数量，会停止流程，不会继续点击最终“添加”，避免库存为 2 但只上架 1 张。
- CSV 导出会包含“是否PSA10”列，方便备份和二次整理。
- 页面直接用 `file://` 打开时，如果浏览器禁止 `sessionStorage`，密码验证会自动退回到当前页面内存状态，不影响进入后台。
- 密码入口带有独立于 React 的兜底表单：即使 React/Babel/CDN 脚本加载失败，也会保留可输入的密码框，避免只显示背景空白页。
- “截图识别新增 -> AI识别”兼容两种接入方式：如果在弹窗里的“AI配置”保存了 OpenAI 兼容直连 API，会优先直连识别；没有直连配置时，会继续使用 Supabase Edge Function `card-screenshot-extract-v1`。
- 直连 API 配置保存在当前浏览器 `localStorage`，默认示例为 `http://43.155.156.14:8080/v1/chat/completions`、模型 `gpt-5.5`、`AI_USE_JSON_SCHEMA=false`。如果页面通过 `file://` 打开，API 网关需要允许浏览器 CORS（常见为 `Origin: null`），否则页面会提示直连被拦截并尝试 Supabase 兜底。
- “AI配置”里的 API Key 输入框默认隐藏，保存状态只显示打码摘要；配置弹窗提供“测试连接”按钮，会用轻量文本请求检查直连 API 是否可用，并显示成功、401/403 或 CORS 拦截等结果。
- Supabase 模式仍然需要在 Edge Function Secrets 里配置 `AI_API_KEY`/`AI_MODEL` 等变量；如果 AI 提供方返回 401/403/429 等错误，页面会在弹窗里保留失败原因，方便检查 API Key、额度、模型权限和接口地址。
- 截图识别结果自动配图时，日版编号会按“卡包代号 + 包内编号”精确匹配：例如 `SV2P-075/071` 会先定位 `SV2P -> Snow Hazard`，再只接受 `Snow Hazard` 里的 `075/071`，避免同编号跨卡包误配图；常见 SV 系列代号会用内置对照表兜底，减少 TCG Collector 卡包列表结构变化带来的漏配。

## 集换社价格每日更新

`scripts/update_jhs_prices.py` 会读取“卡价管理”里状态为“未售出”且带有 `cardCode` 的商品，尝试从本机集换社 App 的只读缓存或可见 UI 中识别“流通品价”和“PSA10价”，然后写回 `jhsRawPrice`、`jhsPsa10Price`、`jhsPriceUpdatedAt` 和 `jhsPriceNote`。

如果同一个编号在宝可梦简中、日文、英文里分别对应不同卡，可以在商品编辑弹窗的“集换社参考价”里设置 `集换社搜索编号` 和 `集换社游戏语言`。每日更新会优先使用这两个字段；未设置时才自动判断。

这个脚本不会解密集换社的 `raw_data`，也不会复制登录态；如果缓存里没有明文价格，会保留原价并记录未更新原因。

Supabase 读取商品列表时会对短暂的 TLS/网络失败进行重试，避免代理或网络节点偶发抖动时直接中断；如果多次重试后仍失败，脚本会退出并保留错误栈，切换网络或代理后可重新运行。
